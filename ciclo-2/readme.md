# 🚀 Ciclo 2 - Ecossistema learnTECH

Bem-vindo ao repositório do **Ciclo 2** da ByteClass. Este ciclo foca na transição do HTML/CSS estático para a lógica de programação dinâmica, manipulação de DOM e arquitetura modular com JavaScript Vanilla.

## 📌 Visão Geral do Ciclo

O Ciclo 2 é estruturado em módulos progressivos, onde cada projeto introduz novos conceitos de engenharia de software, culminando na construção de sistemas modulares e SPAs (Single Page Applications).

## 📂 Projetos no Ciclo

| #   | Projeto             | Descrição                                                         | Stack                 |
| :-- | :------------------ | :---------------------------------------------------------------- | :-------------------- |
| 1   | **Calculadora**     | Manipulação de eventos, operações matemáticas e lógica de estado. | JS Vanilla            |
| 2   | **Formulário**      | Validação de dados, manipulação de inputs e UX de feedback.       | HTML5 / JS            |
| 3   | **Liga da Justiça** | Consumo de dados, filtragem e renderização dinâmica de listas.    | CSS Grid / JS         |

---

## 🏗️ Destaque de Arquitetura: 4-Mini Games

O projeto **Mini Games** serve como o padrão de referência (Gold Standard) deste ciclo, utilizando a **Governança learnTECH**:

- **Core (Router)**: Orquestrador de rotas dinâmicas em `src/core/router.js`.
- **Modularização**: Cada jogo (Forca, Velha, Quebra-Cabeça) é um micro-aplicativo encapsulado.
- **Design Tokens**: Centralização de estilos no `variables.css` com estética Neo-Brutalista.

---

## 🛠️ Tecnologias e Padrões

Neste ciclo, aplicamos rigorosamente os seguintes padrões de mercado:

- **JS Modules (ES6)**: Uso de `export/import` para isolar responsabilidades.
- **Clean Code**: Funções com responsabilidade única e nomes semânticos.
- **BEM (Block Element Modifier)**: Metodologia CSS para manter a especificidade controlada.
- **Neo-Brutalismo UI**: Identidade visual com bordas espessas (`border-thick`) e sombras sólidas (`shadow-flat`).

## ⚖️ Governança ByteClass

1. **Check de BU**: Antes de iniciar qualquer projeto, verifique se está a trabalhar na Business Unit (BU) correta.
2. **Workflow Git**:
   - `main`: Código homologado e funcional.
   - `feature/nome-do-projeto`: Desenvolvimento ativo.
3. **Documentação**: Cada subpasta de projeto deve conter o seu próprio guia técnico de execução.
