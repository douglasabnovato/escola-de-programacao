# 📚 Roteiro de Aula — Trello Mini Kanban

## 🎯 Objetivos da Aula

- Ensinar como criar um quadro Kanban simples com **HTML, CSS e JavaScript**.
- Trabalhar conceitos de **usabilidade**, **design responsivo** e **interatividade**.
- Mostrar boas práticas de documentação com **Markdown**.
- Estimular pensamento crítico sobre **UX e produto digital**.

---

## 🕒 Estrutura da Aula (2h30)

### 1. Introdução (15 min)

- Apresentar o conceito de **Kanban** e sua aplicação em gestão de tarefas.
- Mostrar exemplos reais (Trello, Jira).
- Explicar o que será construído: um **mini Kanban com três colunas**.

---

### 2. Estrutura HTML (25 min)

- Criar o arquivo `index.html`.
- Montar o **header** com título e subtítulo.
- Estruturar as três colunas: **À Fazer, Fazendo e Feito**.
- Inserir botões para adicionar tarefas.

**Atividade prática:** cada aluno monta a estrutura básica no seu editor.

---

### 3. Estilo com CSS (30 min)

- Aplicar cores, fontes e responsividade.
- Criar animações simples (fadeIn, hover).
- Estilizar os cards e o footer.

**Atividade prática:** alunos personalizam cores e fontes para dar identidade própria.

---

### 4. Interatividade com JavaScript (40 min)

- Implementar o **modal** para adicionar tarefas.
- Criar funções para adicionar e remover cards.
- Implementar **drag & drop** para mover tarefas entre colunas.

**Atividade prática:** alunos testam adicionar, mover e excluir tarefas.

---

### 5. Footer Avançado (20 min)

- Estruturar três colunas:
  - **Esquerda:** logo + nome + tagline.
  - **Centro:** links externos.
  - **Direita:** redes sociais.
- Implementar animações e responsividade.

**Atividade prática:** alunos inserem suas próprias logos e links.

---

### 6. Documentação com Markdown (20 min)

- Explicar o que é um arquivo `README.md`.
- Mostrar como escrever títulos, listas, blocos de código.
- Criar o README para o projeto com:
  - Descrição
  - Funcionalidades
  - Tecnologias
  - Estrutura de pastas
  - Como executar
  - Próximos passos
  - Licença

**Atividade prática:** cada aluno escreve seu próprio `README.md`.

---

### 7. Encerramento e Reflexão (10 min)

- Revisar o que foi aprendido.
- Discutir como aplicar esse conhecimento em projetos reais.
- Destacar importância de **UX e documentação** em produtos digitais.

---

## 📌 Materiais Necessários

- Editor de código (**VS Code recomendado**).
- Navegador atualizado.
- Repositório GitHub para versionamento.
- Logo/imagem para personalização do footer.

---

## 🎓 Resultado Esperado

Ao final da aula, cada aluno terá:

- Um **mini Kanban funcional** com HTML, CSS e JS.
- Um **footer avançado** com logo, links e redes sociais.
- Um **README.md completo** documentando o projeto.
- Experiência prática em **UX aplicado ao front‑end**.
